# 🌐 Laboratório de Redes

Este projeto apresenta estudos e testes práticos com comandos de rede utilizados para diagnóstico de conectividade em computadores.

O objetivo é demonstrar conhecimentos básicos de troubleshooting de rede e análise de conexão.

---

## 🎯 Objetivo

Aprender a diagnosticar problemas de rede utilizando comandos disponíveis no sistema operacional.

---

## 🧪 Testes realizados

### PING

O comando `ping` é utilizado para verificar se um dispositivo consegue se comunicar com outro na rede.

Exemplo:

ping google.com

O comando envia pacotes para o servidor e mede o tempo de resposta.

Informações obtidas:

- Tempo de resposta da rede
- Perda de pacotes
- Conectividade com o destino

---

### TRACERT

O comando `tracert` mostra o caminho que os pacotes percorrem até chegar ao destino.

Exemplo:

tracert google.com

Esse comando permite identificar:

- Quantos roteadores existem no caminho
- Onde pode estar ocorrendo lentidão
- Possíveis falhas na rota da rede

---

### IPCONFIG

O comando `ipconfig` mostra as configurações de rede do computador.

Exemplo:

ipconfig

Informações exibidas:

- Endereço IP
- Gateway padrão
- Servidores DNS
- Máscara de sub-rede

---

## 🛠 Ferramentas utilizadas

- Prompt de comando do Windows
- Rede local
- Conexão com internet
